功能概述

12CM 的 Admin Plugin 透過 LINE OA Plus TW SDK（dx 物件）取得當前登入使用者的商戶資訊、Plugin 狀態及角色，並根據這些資訊動態調整 UI 語言、功能可見範圍與操作權限，確保 OA Plus 與 12CM 系統之間的角色體驗一致。

ℹ️
本文件範疇 涵蓋：SDK 載入、前端 API 用法、角色判斷邏輯、CSRF 安全機制、DX-JWT 後端驗證、API 請求路由規則，以及 UI/UX、沙盒、CSP 等實作要點。

角色控管 LINE 規格 內部邏輯

6.1 LINE 規範的角色邊界 LINE 規格

LINE OA Plus 定義兩種角色，僅規範「訂閱與帳務」相關操作的權限邊界，其他功能的權限由 12CM 自行決定。

Admin　管理者
訂閱與帳務操作的全部權限
  • ✓發起訂閱付費版
  • ✓更新付款方式
  • ✓取消 / 恢復訂閱
  • ✓查看訂閱清單與扣款紀錄
Operator　操作員
訂閱與帳務僅可查看，不可變更
  • ✓查看訂閱清單
  • ✓查看訂閱狀態
  • ✓查看扣款紀錄
  • ✗發起訂閱
  • ✗修改付款資料
ℹ️
LINE 只提供角色資訊，不規範 12CM 功能邊界 Product Guide §5 說明：LINE 透過 SDK 提供角色資訊「供 Partner 參考」，Partner 可自行決定如何對應至自身系統的權限設計。訂閱與帳務以外的功能（如商品管理、訂單查看等），LINE 沒有限制，由 12CM 自行定義 Admin 與 Operator 各自能操作的範圍。
⚠️
角色驗證前後端都需實作 前端負責根據角色控制 UI 可見性與操作入口（show/hide）；後端 API 同樣需驗證角色，防止 Operator 繞過前端直接呼叫受限 API。兩層驗證缺一不可。

dx.getUser() — 商戶資訊 LINE 規格

回傳欄位

欄位類型說明
merchantId String 商戶 ID（例如 @abc1234）
language String 使用者語系，支援：zh-TW、en、ja、ko、th、id

使用範例

dx.getUser()
  .then((user) => {
    const merchantId = user.merchantId;
    const language   = user.language;
    // 根據 language 切換 UI 語系
    i18n.setLanguage(language);
  })
  .catch((err) => {
    console.error('getUser error:', err.code, err.message);
  });

錯誤處理

Error Code說明
TIMEOUTSDK 在 5,000 ms 內未收到 OAP 回應
EXCEPTION非預期 runtime 錯誤

dx.getPluginInfo() — Plugin 狀態 LINE 規格

回傳欄位

欄位類型可能值說明
level String FREE / PREMIUM 目前方案等級
activatedStatus String ACTIVATED / ACTIVATION_INITIATED / ACTIVATION_FAILED Plugin 啟用狀態

使用範例

const pluginInfo = dx.getPluginInfo();
const level  = pluginInfo.level;            // "FREE" | "PREMIUM"
const status = pluginInfo.activatedStatus;   // "ACTIVATED" | ...

if (level === 'PREMIUM' && status === 'ACTIVATED') {
  showPremiumFeatures();   // 顯示付費功能區塊
} else {
  showUpgradePrompt();     // 顯示升級提示
}
ℹ️
getPluginInfo() 為同步呼叫，不需要 .then()，可在頁面初始化時直接使用。

CSRF 安全機制 LINE 規格

所有非 GET 請求（POST / PUT / DELETE）必須帶入 CSRF Token：

const csrfToken      = dx.security.getCSRFToken();
const csrfHeaderName = dx.security.getCSRFTokenHeaderName(); // "DX-CSRF-TOKEN"

fetch('/api/your-endpoint', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    [csrfHeaderName]: csrfToken
  },
  body: JSON.stringify(payload)
});
⚠️
缺少 CSRF Token 的非 GET 請求將被拒絕。每次操作前都需重新取得最新 Token，不應快取舊 Token。
8

DX-JWT 後端驗證 LINE 規格

OA Plus 反向代理會在每個請求的 cookie 中注入 dx-jwt，12CM 後端須驗證此 JWT。

JWT 結構

Header
{ "alg": "HS256", "typ": "JWT" }
Payload
{
  "pluginId": "12345",
  "merchantId": "123456",
  "pluginInfo": {
    "level": "FREE",
    "activatedStatus": "ACTIVATED"
  },
  "iat": 1717023000,
  "exp": 1717026600
  // role 欄位是否存在待 LINE 技術窗口確認
}

驗簽方式

使用 LINE 提供的 Secret of Admin Plugin 以 HMAC-SHA256 驗簽。

項目說明
演算法HMAC-SHA256
Secret 來源LINE 核發的 Secret of Admin Plugin（詳見 Phase 2 對接清單）
Token 有效期60 分鐘（exp 欄位）
注入方式反向代理自動注入至 dx-jwt cookie

API 請求路由規則 LINE 規格

  • 所有 API 請求必須透過反向代理發送，不得直接呼叫 12CM 後端域名
  • 請求 URL 必須使用與 Plugin 頁面相同的 domain
// Plugin 頁面 URL
https://testplugin-dx-staging.line-apps.com/

// API 請求 URL（同 domain）
https://testplugin-dx-staging.line-apps.com/api/your-endpoint
ℹ️
身份驗證依賴 plugin domain 上的 dx-jwt cookie，因此 API 呼叫必須透過 plugin domain，不能直接打 12CM 後端 domain。

3xx 重定向限制

HTTP 狀態碼反向代理行為
304 Not Modified✅ 允許通過
其他 3xx❌ 封鎖，前端收到 502 Bad Gateway
⚠️
若服務設計依賴 3xx redirect（例如登入導頁），須改為前端 JS 主動導向或 API 回傳 redirect URL 的方式實作。

iFrame 沙盒限制 LINE 規格

Admin Plugin 在 iFrame 中執行，sandbox 允許以下能力：

allow-same-origin 允許同源 cookie、localStorage、IndexedDB 操作
allow-scripts 允許執行 JavaScript 腳本
⚠️
Sandbox 未列出的能力（如 allow-forms、allow-popups）均不可使用，設計 UI 時需注意。

CSP 合規要求 LINE 規格

12CM Plugin 頁面須遵守平台的 Content Security Policy。LINE 強制執行的 CSP 指令如下：

指令限制
script-src'self' + 白名單可信來源
style-src'self' 'unsafe-inline' + 白名單可信來源
img-src'self' + 白名單可信來源
media-src'self'（同源）
font-src'self' + 白名單可信來源
  • 動態注入的 <script> 與 <link> 標籤須包含有效 nonce，否則將被 CSP 封鎖
  • 所有外部資源須在 ACL（Access Control List）中明確定義，未列入的資源將被封鎖
